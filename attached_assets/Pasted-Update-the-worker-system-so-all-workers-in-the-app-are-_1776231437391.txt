Update the worker system so all workers in the app are treated as subcontractors, not employees.

Business rule:
- all workers are subcontractors
- remove the need to distinguish employee vs contractor
- every worker should be able to generate a proper invoice PDF based on real worked hours tracked with Start / Stop

MAIN GOAL
Create a subcontractor invoice system inside the app, where each subcontractor can fill in their own business and bank details in their profile, then generate a professional PDF invoice from tracked hours and share/send it easily.

1. WORKER ROLE / TERMINOLOGY
- Replace employee terminology with subcontractor terminology where appropriate in the worker-facing experience
- Keep admin functionality working
- Do not break existing login, permissions, or time tracking
- Admin still creates subcontractor accounts
- Subcontractor logs in and manages their own profile and invoices

2. SUBCONTRACTOR PROFILE FIELDS
Add these editable profile fields for each subcontractor:
- full name
- email
- phone
- ABN
- business address (optional)
- bank name
- account name
- BSB
- account number
- invoice notes (optional)

Requirements:
- subcontractor must be able to edit these fields in their own profile
- keep the profile page mobile-friendly and well organized
- split the profile form into clear sections:
  - Personal Details
  - Business Details
  - Payment Details
  - Invoice Notes

3. ADMIN SETTINGS FOR INVOICE BRANDING
Allow admin to configure company invoice/business details used on generated invoices:
- company name
- company logo
- ABN / tax number
- business email
- business address

Requirements:
- these settings should be used automatically in all subcontractor invoices
- keep existing company branding/settings working

4. NEW SUBCONTRACTOR SECTION: "MY INVOICES"
Add a new screen/section for subcontractor users called:
- My Invoices

This screen should allow the subcontractor to:
- choose a period:
  - This Week
  - Last Week
  - Custom Date Range
- see a summary before generating:
  - projects worked
  - total hours worked
  - hourly rate
  - total amount
- generate a PDF invoice
- share/export the PDF
- open native share flow and email/share options with the PDF

If direct email attachment is complex, first implement robust native share/export support for the generated PDF.

5. DATA SOURCE / CALCULATION RULES
The invoice must be generated only from real tracked worked hours already recorded by the existing Start / Stop time tracking system.

Requirements:
- use actual stored work sessions only
- calculate total hours from selected date range
- use the subcontractor's hourly rate already stored in the app
- calculate total invoice amount automatically
- support project breakdown when multiple projects were worked in the selected period
- do not allow manual fake hour entry on the invoice screen

Generate a clean professional PDF invoice that includes:

Header:
- company logo

- "INVOICE" title

Company details:
- company business email
- company business address
- company ABN / tax number

Subcontractor details:

Totals:

Payment details section:
- bank name
- account name

Footer:
- optional invoice notes
- clean professional closing layout

Add automatic invoice numbering.

Requirements:

Design the PDF invoice in a clean, professional Australian subcontractor style.
